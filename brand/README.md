# Brand System Quick Start
## The `<mdp>` Mark

---

## Core Concept

The `<mdp>` mark uses chevron brackets to encode the fundamental rhythm of design thinking:

| Symbol | Meaning |
|--------|---------|
| `<` | **Diverge** — explore, open up, generate options |
| `>` | **Converge** — synthesize, focus, make decisions |
| `<>` | **Full cycle** — one complete iteration |
| `<<>>` | **Rapid iteration** — sprint mode |

---

## Quick Usage

### Logo
- Primary: `<mdp>` in JetBrains Mono
- Always lowercase
- Minimum clear space: height of the `m` on all sides

### Typography
- **Primary font:** JetBrains Mono
- **Fallback:** Any monospace font
- [Full specs →](specs/typography.md)

### Colors (v1)
- **Current:** Monochrome only
- **Primary:** Near-black for text, white for backgrounds
- **Avoid:** UVA Navy (#232D4B), UVA Orange (#E57200)
- [Full specs →](specs/color.md)

## Character System

### Spark `<•>`
The course "voice" — appears in:
- Slide transitions
- Terminal output
- Explainer moments

Use 2-3 times per deck maximum.

### Kaomoji (Rare)
Reserved for empathy moments only. Approved set:
- `¯\_(ツ)_/¯` — "It happens"
- `ʕ•ᴥ•ʔ` — Friendly curiosity

Use 2-3 times total across entire course.

## When in Doubt

1. Keep it monochrome
2. Use JetBrains Mono
3. Less is more with characters
4. The brackets are the brand — don't over-decorate
